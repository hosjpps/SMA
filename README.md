# SURIKOV Marketing Agency — лендинг

Одностраничный сайт маркетингового агентства (CMO на аутсорсе для малого и среднего бизнеса) с формой заявки, которая отправляет лиды в Telegram.

## Возможности

- Многосекционный лендинг: hero, боли клиента, решение, услуги, подход, кейсы, форматы работы, консультация, о компании, процесс, FAQ и контакты
- Форма заявки на консультацию: имя, ниша бизнеса, сайт/соцсети, контакт, комментарий
- Серверный API-роут, который принимает заявку и отправляет её сообщением в Telegram-чат
- Плавный скролл к секциям по якорным ссылкам (в том числе при переходе с других страниц)
- Cookie-баннер и отдельные страницы правовых документов: политика cookie, политика обработки персональных данных, политика конфиденциальности
- Адаптивная вёрстка и переключение темы (next-themes)
- Подключённая веб-аналитика (Vercel Analytics)

## Стек

- Next.js 16 (App Router) + React 19
- TypeScript
- Tailwind CSS v4
- UI-компоненты на Radix UI (shadcn-style), иконки lucide-react
- react-hook-form + zod для форм и валидации
- recharts, embla-carousel, sonner и другие UI-утилиты
- Шрифты Google Fonts (Dela Gothic One, Montserrat)

## Структура

```
app/
  page.tsx                 # сборка лендинга из секций
  layout.tsx               # метаданные, шрифты, аналитика
  globals.css
  api/telegram/route.ts    # приём заявки и отправка в Telegram
  cookie-policy/           # страница политики cookie
  personal-data-policy/    # обработка персональных данных
  privacy-policy/          # политика конфиденциальности
components/                # секции лендинга (*-section.tsx) и UI-компоненты
  ui/                      # базовые компоненты (shadcn-style)
hooks/                     # пользовательские React-хуки
lib/                       # вспомогательные функции
public/                    # статика
styles/                    # стили
```

## Запуск

```bash
# установка зависимостей (используется pnpm)
pnpm install

# дев-сервер
pnpm dev          # http://localhost:3000

# продакшен-сборка
pnpm build
pnpm start
```

Форма заявки шлёт сообщения в Telegram через `app/api/telegram/route.ts`. Токен бота и ID чата стоит вынести в переменные окружения (`TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`) и читать их из `process.env`, а не хранить в коде.
